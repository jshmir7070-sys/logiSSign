# logiSSign 정보보호 정책서
> 버전: 1.0 | 시행일: 2026-04-04 | 관리자: 플랫폼 운영팀

---

## 1. 목적
본 정책은 logiSSign 플랫폼에서 처리하는 모든 정보자산의 기밀성, 무결성, 가용성을 보장하기 위한 정보보호 관리 기준을 수립한다.

## 2. 적용 범위
- logiSSign 웹 포탈 (logissign.com)
- logiSSign 모바일 앱 (iOS/Android)
- Supabase 데이터베이스 및 스토리지
- 외부 연동 서비스 (PortOne, Solapi, Resend, Sentry)

## 3. 정보보호 원칙
1. 최소 권한 원칙: 업무 수행에 필요한 최소한의 접근 권한만 부여
2. 직무 분리: 관리자/대리점/기사 역할을 분리하여 권한 통제
3. 개인정보 암호화: PII 필드는 AES-256-GCM으로 암호화 저장
4. 전송 구간 암호화: 모든 통신은 TLS 1.2+ 적용

## 4. 접근 통제
### 4.1 역할 기반 접근 제어 (RBAC)
| 역할 | 접근 범위 |
|------|----------|
| provider_admin | 전체 시스템 관리 |
| agency_admin | 소속 대리점 데이터만 |
| driver | 본인 데이터만 |

### 4.2 인증 체계
- 1단계: 이메일 + 비밀번호 (8자 이상, 대소문자+숫자+특수문자)
- 2단계: SMS OTP (6자리, 5분 만료, 최대 5회 시도)
- 세션: MFA 쿠키 (HMAC-SHA256, 세션 쿠키, 24시간 TTL)

### 4.3 데이터베이스 접근 제어
- Row Level Security (RLS) 123개 정책 적용
- 직접 DB 접근 차단 (API 통해서만 접근)
- Service Role Key는 서버사이드에서만 사용

## 5. 개인정보 보호
### 5.1 수집 항목
| 구분 | 필수 | 선택 |
|------|------|------|
| 대리점 | 이메일, 비밀번호, 사업자번호, 대표자명, 전화번호 | 주소, 계좌정보, 마케팅 동의 |
| 기사 | 이름, 전화번호 | 생년월일, 계좌정보, 배송구간 |

### 5.2 암호화 대상
- 계좌번호: AES-256-GCM 암호화
- 전화번호: AES-256-GCM 암호화
- 생년월일: AES-256-GCM 암호화
- 비밀번호: bcrypt 단방향 해시

### 5.3 보유 기간
| 데이터 | 보유 기간 | 근거 |
|--------|----------|------|
| 계약서 | 10년 | 전자서명법 |
| 정산 기록 | 5년 | 국세기본법 |
| 로그인 기록 | 3개월 | 통신비밀보호법 |
| 탈퇴 후 개인정보 | 30일 내 파기 | 개인정보보호법 |

## 6. 감사 로그
- security_logs: 인증, 권한, PII 접근 등 9개 이벤트 유형
- pii_audit_log: 개인정보 접근/조회 기록
- plan_change_log: 플랜 변경 감사 추적
- contract_signatures.audit_log: 전자서명 감사 추적

## 7. 취약점 관리
- 분기 1회: 의존성 보안 스캔 (npm audit)
- 분기 1회: Supabase 보안 어드바이저 점검
- 연 1회: 모의 침투 테스트

## 8. 백업 및 복구
- Supabase 자동 일일 백업 (30일 보존)
- Point-in-Time Recovery (PITR) 지원
- RTO: 4시간 / RPO: 24시간

## 9. 수탁자 관리
| 수탁자 | 위탁 업무 | 보유 데이터 |
|--------|----------|------------|
| Supabase (AWS ap-northeast-2) | DB/인증/스토리지 | 전체 |
| Vercel | 웹 호스팅 | 세션 데이터 |
| PortOne (KCP) | 결제 처리 | 빌링키 (토큰) |
| Solapi | SMS 발송 | 전화번호 (일시적) |
| Resend | 이메일 발송 | 이메일 주소 (일시적) |
| Sentry | 에러 모니터링 | 에러 로그 (PII 제외) |
