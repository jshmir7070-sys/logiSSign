## 2. CRITICAL 이슈 (즉시 수정 필요)

### 2-1. Rate Limiter 분산 환경 미지원
- **파일:** `web/src/lib/rate-limit.ts`
- **문제:** 인메모리 Map 기반 → Vercel 다중 인스턴스에서 각각 별도 카운트
- **영향:** API 브루트포스 공격 가능
- **해결:** Upstash Redis 기반 분산 Rate Limiter로 교체

### 2-2. CRON 인증 개발환경 바이패스
- **파일:** `web/src/lib/api-auth.ts:115-123`
- **문제:** CRON_SECRET 미설정 시 개발환경에서 인증 없이 통과
- **해결:** 모든 환경에서 CRON_SECRET 필수 적용

### 2-3. 기사 가입 API 입력 검증 미흡
- **파일:** `web/src/app/api/auth/driver-signup/route.ts`
- **문제:** Zod 스키마 없이 직접 JSON 파싱, 비밀번호 복잡성 미검증
- **해결:** signup route와 동일한 Zod 스키마 적용

### 2-4. 파일 업로드 MIME 타입 미검증
- **파일:** `web/src/app/api/documents/upload/route.ts`
- **문제:** 클라이언트 제공 MIME 타입만 확인, 실제 파일 내용 미검증
- **해결:** file-type 라이브러리로 서버사이드 MIME 검증

## 6. 전체 평가

**강점:**
- RLS 정책 123개 — 데이터 격리 우수
- CSRF 보호, timing-safe 인증 체크 등 보안 기반 탄탄
- Sentry 모니터링 이미 구축

**약점:**
- 분산 환경 고려 부족 (Rate Limiter, 세션)
- 런타임 타입 검증 미흡 (as unknown 캐스트 남용)
- 에러 추적 컨텍스트 부족

**결론:** 핵심 아키텍처는 안정적이나, Critical 4건과 High 7건 수정 후 프로덕션 출시 가능. 예상 수정 기간: 1-2주.
