중간 Rate limiting 없음 — /api/verify, /api/integrity-check 등 공개/관리 API에 요청 제한 미적용.

다음 단계 추천 (우선순위)

1️⃣ Supabase 타입 생성 + AnyQuery 제거

npx supabase gen types typescript 실행 후 Database 타입을 적용하면, 현재 21개 서비스의 as AnyQuery 캐스팅을 모두 제거하고 컴파일 타임 타입 안전성을 확보할 수 있습니다. 가장 적은 비용으로 코드 품질을 크게 올리는 작업입니다.

2️⃣ 나머지 테이블 RLS 보완

providers, agencies, contract_templates, notices, subscriptions에 RLS 정책을 추가하여 서비스 키 없이도 데이터 격리가 보장되도록 합니다. 특히 agencies는 타 대리점 데이터 접근 차단이 필수입니다.

3️⃣ API 입력 검증 (Zod)

모든 POST 엔드포인트에 Zod 스키마를 적용하여 유효하지 않은 입력을 사전 차단합니다. 특히 /api/contracts/send, /api/verify, /api/integrity-check 가 우선입니다.

4️⃣ DB 트랜잭션 적용

계약 생성(contracts + contract_signatures), 서명 완료(update contracts + generate PDF + audit) 같은 다중 테이블 작업을 PostgreSQL Function 또는 Supabase Edge Function에서 트랜잭션으로 감싸야 합니다.

5️⃣ 위변조 감지 알림 연동

integrity-check CRON에서 실패 건 발견 시 Slack Webhook 또는 관리자 이메일로 즉시 알림을 보내도록 합니다. 현재는 console.error만 출력되어 실시간 대응이 불가능합니다.
